# سند جامع معماری هفت‌لایه پلتفرم هوش مصنوعی سازمانی آفلاین

## ۱) مقدمه و فلسفه طراحی

این سند، نسخه‌ی یکپارچه و غیرتکراری دو گزارش فنی موجود است و معماری پیشنهادی برای استقرار هوش مصنوعی در محیط‌های **Air-gapped** را ارائه می‌دهد. در چنین محیط‌هایی، مالکیت کامل داده، پایداری عملیاتی، و قابلیت حسابرسی، الزامات پایه هستند.

الگوی طراحی بر ترکیب دو رویکرد استوار است:

- **معماری لایه‌ای مدولار (Modular Layered):** برای تفکیک روشن مسئولیت‌ها و مهار خطا در سطح لایه.
- **معماری شش‌وجهی (Hexagonal / Ports & Adapters):** برای استقلال از فناوری‌های زیرساختی و امکان تعویض اجزا بدون بازنویسی منطق هسته.

این ترکیب، اصول کلیدی زیر را تضمین می‌کند: **Separation of Concerns**، **Loose Coupling**، **Backend Agnosticism**، **Fault Containment** و **Rebuild-from-source**.

---

## ۲) مزایای راهبردی معماری

### ۲.۱ چابکی تکنولوژیک با اتصال سست
با تفکیک لایه‌ها، تغییر مدل زبانی (مثلا از Llama به Mistral) یا جایگزینی موتور OCR، بدون اختلال در کل سامانه انجام می‌شود.

### ۲.۲ مقیاس‌پذیری افقی مبتنی بر صف
در محیط آفلاین و CPU-centric، رشد بار پردازشی از طریق افزایش Workerها و صف‌محور کردن پردازش مدیریت می‌شود؛ نه با ارتقای عمودی پرهزینه.

### ۲.۳ رویت‌پذیری، حاکمیت و امنیت
ثبت کامل رخدادها، رهگیری انتهابه‌انتها و کنترل دسترسی سندمحور، ریسک عملیاتی و امنیتی را کاهش می‌دهد و ممیزی‌پذیری سازمانی را حفظ می‌کند.

---

## ۳) نمای کلان جریان داده

معماری در دو مسیر اصلی عمل می‌کند:

- **Write Path:** تبدیل فایل خام به دانش قابل جستجو (Extraction, Normalization, Embedding, Indexing).
- **Read Path:** تبدیل پرسش کاربر به پاسخ مستند (Query Understanding, Retrieval, Reranking, Generation).

هفت لایه عملیاتی این دو مسیر را پشتیبانی می‌کنند.

---

## ۴) تشریح فنی لایه‌های هفت‌گانه

### لایه ۱: ورود داده (Ingestion / Gatekeeper)

وظایف اصلی:

- اعتبارسنجی ورودی (MIME Type Check).
- پاک‌سازی و امنیت اولیه (Sanitization + اسکن آفلاین با **ClamAV**).
- جلوگیری از پردازش تکراری با **Idempotency** بر اساس هش **SHA-256**.
- کنترل Back-pressure با **RabbitMQ** یا **Redis**.

خروجی این لایه، بسته‌ی استانداردشده‌ی پردازش برای مراحل بعدی است.

### لایه ۲: پردازش و پالایش (Processing / Refinery)

این لایه مسئول استخراج دقیق و آماده‌سازی زبان‌محور محتوا است:

- **Smart Router** برای انتخاب مسیر پردازش:
  - اسناد متنی: ابزارهای سریع مانند **PyMuPDF**.
  - اسناد پیچیده یا چیدمان‌محور: **Docling / Surya**.
  - اسناد اسکن‌شده: OCR (مانند Tesseract/PaddleOCR یا موتورهای معادل).
- **Persian Normalization** با **Hazm** و **DadmaTools**:
  - یکسان‌سازی حروف فارسی/عربی.
  - مدیریت نیم‌فاصله (**ZWNJ**).
  - حذف نویزهای کاراکتری.
- **Chunking** با روش **Recursive** و رعایت مرز جمله، همراه با همپوشانی حدود ۲۰٪ (تقریبا ۵۰ تا ۱۰۰ توکن) برای حفظ پیوستگی معنایی.

### لایه ۳: بازنمایی معنایی (Embedding / Semantic Translator)

- تبدیل چانک‌ها به بردار معنایی با مدل‌های چندزبانه/فارسی‌محور (مانند **E5**, **BGE-M3**, **LaBSE**).
- بهینه‌سازی CPU با تبدیل مدل‌ها به **ONNX** و کوانتیزاسیون **INT8** روی **ONNX Runtime**.
- استفاده از **Batch Processing** برای کاهش سربار و افزایش توان عملیاتی.

### لایه ۴: حافظه و ایندکس ترکیبی (Hybrid Index / Memory)

ترکیب سه سازوکار ذخیره‌سازی:

- **Vector Store:** مانند **Qdrant** (HNSW) برای جستجوی معنایی.
- **Inverted Index:** مانند **BM25** در OpenSearch برای انطباق دقیق واژه‌ها، کدها و شناسه‌های سازمانی.
- **Metadata Store:** مانند **PostgreSQL** (یا SQLite در مقیاس کوچک) برای متادیتا، فیلترها و سیاست‌های دسترسی.

قابلیت‌های تکمیلی:

- حذف نرم با **Tombstone** جهت حفظ ردپای حسابرسی.
- **Index Version Registry** برای مدیریت نسخه‌ی مدل/ایندکس و امکان Rollback.
- کاهش افزونگی با Dedup چندمرحله‌ای (LSH + خوشه‌بندی، آستانه شباهت کسینوسی بالا).

### لایه ۵: ارکستراتور RAG (Query Pipeline / Brain)

فرآیند پاسخ‌سازی:

1. **Query Transformation:** بازنویسی و توسعه پرسش با فرهنگ مترادف محلی.
2. **Hybrid Retrieval:** بازیابی همزمان معنایی و کلیدواژه‌ای.
3. **Score Fusion:** ادغام رتبه‌ها با **RRF**.
4. **Context Assembly:** ساخت زمینه کنترل‌شده برای مدل.
5. **Generation:** تولید پاسخ با LLM بومی (مثل نسخه‌های GGUF فارسی) با دستورالعمل سخت‌گیرانه ضد توهم.

### لایه ۶: ارائه سرویس (API / Interface)

- ارائه API از طریق **REST** یا **gRPC**.
- پشتیبانی از **SSE Streaming** برای پاسخ‌دهی تدریجی.
- اعمال **JWT**, **Rate Limiting** و سیاست‌های دسترسی برای ایمنی لایه سرویس.

### لایه ۷: نظارت و حاکمیت (Observability / Watchtower)

- **Audit Logs** غیرقابل تغییر برای ممیزی.
- **Tracing** انتهابه‌انتها (مانند Jaeger) برای عیب‌یابی.
- مانیتورینگ منابع با **Prometheus + Grafana** و تجمیع لاگ با **Loki**.
- **Data Lineage** برای رهگیری کامل مسیر پاسخ از فایل منبع تا خروجی.
- **Feedback Loop** برای بهبود کیفیت.
- ارزیابی دوره‌ای **Embedding Drift** (مثلا با KS-test).

---

## ۵) ملاحظات تخصصی زبان فارسی

برای اسناد فارسی، وابستگی صرف به Vector Search کافی نیست. ترکیب جستجوی برداری با **BM25** برای بازیابی دقیق اسامی خاص، کدهای اداری و شماره‌ها ضروری است. همچنین کیفیت بازیابی و پاسخ‌گویی به شدت به نرمال‌سازی درست نیم‌فاصله، نویسه‌ها و علائم نگارشی وابسته است.

---

## ۶) پشته‌ی پیشنهادی ابزارها (Offline-Ready Stack)

| حوزه | ابزار پیشنهادی | دلیل انتخاب در محیط آفلاین/فارسی |
| --- | --- | --- |
| پردازش سند | Docling / Surya / PyMuPDF | استخراج ساختارمند و پوشش سناریوهای اسکن و متن |
| OCR | Tesseract / PaddleOCR / معادل سازمانی | پوشش اسناد تصویرمحور و دوزبانه |
| NLP فارسی | Hazm / DadmaTools | نرمال‌سازی و پردازش ریخت‌شناختی فارسی |
| Embedding | E5 / BGE-M3 / LaBSE | تعادل کیفیت معنایی در سناریوهای فارسی-چندزبانه |
| Vector DB | Qdrant (HNSW) | کارایی و فیلترینگ مناسب متادیتا |
| Search Engine | OpenSearch (BM25 + ICU) | تطبیق دقیق واژه و سازگاری بهتر با RTL/ZWNJ |
| Object Storage | MinIO | نگهداری امن فایل خام در محیط ایزوله |
| Workflow | Celery / Temporal | مدیریت پردازش‌های طولانی و بازیابی از خطا |
| LLM Inference | llama.cpp | اجرای مدل‌های کوانتایز شده روی CPU |
| Observability | Prometheus / Grafana / Loki / Jaeger | رویت‌پذیری کامل عملیاتی |

---

## ۷) راهبرد استقرار و توسعه‌پذیری

- شروع با **Single-node** برای سادگی عملیاتی.
- توسعه تدریجی به معماری توزیع‌شده با افزایش Worker Pool.
- حفظ پایداری از طریق نسخه‌بندی ایندکس، خط‌مشی حذف نرم، و مانیتورینگ مستمر کیفیت.

---

## ۸) جمع‌بندی

این معماری هفت‌لایه، چارچوبی عملیاتی و پایدار برای پلتفرم هوش مصنوعی سازمانی آفلاین فراهم می‌کند: از یک‌سو با تفکیک لایه‌ها، تغییرپذیری فناوری را ممکن می‌سازد؛ و از سوی دیگر با حاکمیت داده، امنیت و رویت‌پذیری عمیق، الزامات سخت‌گیرانه‌ی محیط‌های ایزوله را برآورده می‌کند. خروجی نهایی، سامانه‌ای است که هم در مقیاس سازمانی قابل اتکاست و هم در برابر تغییرات سریع فناوری، انعطاف‌پذیر باقی می‌ماند.
